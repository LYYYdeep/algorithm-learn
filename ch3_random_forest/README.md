# Random Forest Notes

## 核心思想

随机森林是多棵决策树组成的集成模型。

```text
Random Forest = Bagging Decision Trees + Feature Randomness
```

分类任务：多棵树投票。

回归任务：多棵树预测值平均。

## 随机性来源

### 1. 样本随机

每棵树使用 bootstrap 有放回采样得到的训练子集。

### 2. 特征随机

每个节点分裂时，只随机选择部分特征作为候选特征。

## Bagging

Bagging = Bootstrap Aggregating。

流程：

```text
1. 通过 bootstrap 有放回采样生成多个训练子集
2. 每个子集训练一个基模型
3. 聚合多个模型结果
```

分类聚合：投票。

回归聚合：平均。

## 为什么比单棵树稳定

单棵树方差大，对数据扰动敏感。

随机森林通过多棵树投票/平均降低方差，提高稳定性和泛化能力。

## 常用参数

- `n_estimators`：树的数量。树越多通常越稳定，但训练和预测更慢。
- `max_features`：每个节点分裂时考虑的特征数。用于降低树之间相关性。
- `max_depth`：每棵树最大深度。
- `bootstrap=True`：使用有放回采样。
- `oob_score=True`：使用 OOB 样本估计性能。

## OOB

OOB = Out-of-Bag，袋外样本。

对于某一棵树，bootstrap 采样时没有被采到的训练样本，就是这棵树的 OOB 样本。

OOB score：用没有见过某个样本的树来预测该样本，再汇总得到内部验证分数。

OOB 可以作为验证估计，但不能完全替代最终独立测试集。

## Feature Importance

`feature_importances_` 通常基于不纯度下降。

一个特征越常用于重要分裂，并带来更大 Gini/Entropy 下降，重要性越高。

注意：

- 特征重要性之和通常为 1
- 只能说明相关性或模型依赖，不代表因果关系
- 基于不纯度下降的重要性可能偏向连续值多、类别基数高的特征

## 面试重点

### 随机森林的随机性来自哪里？

样本随机和特征随机。

### Bagging 决策树和随机森林区别？

Bagging 决策树主要只有样本随机；随机森林在此基础上加入节点分裂时的特征随机。

### 随机森林需要标准化吗？

通常不需要，因为底层是树模型，基于特征阈值和排序关系分裂。
